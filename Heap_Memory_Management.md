# 배경 개념

- 세그먼트(영역, segment): 주소 공간 중에서 특정 역할을 하는 영역. 대표적으로 스택/힙/데이터/코드 영역이 있다. 
- 힙 영역(heap segment): 프로그래머가 완전히 직접 제어할 수 있는 큰 메모리 영역
- 힙 할당기: 전문적으로 힙 영역의 메모리를 할당하고 해제하는 작은 프로그램
  - 왜 필요한가?: 추상화, 표준화, 여러 환경이나 OS에서 일관적으로 돌아갈 수 있음.

# 힙 메모리 관리 (Heap Memory Management)

동적 메모리 할당은 실행 시간에 변하는 메모리 수요를 처리할 수 있게 해준다. 메모리를 할당하고/회수하여 효율적으로 메모리 수요를 처리한다.

1. **블록 표현 (Representation)**: 힙 메모리를 어떤 블록 단위로 나누고, 각 블록에 어떤 정보를 둘 것인가
2. **빈 공간 추적 (Tracking)**: 사용 가능한 free block을 어떤 방식으로 관리할 것인가
3. **할당/해제 정책 (Policy)**: 어떤 블록을 선택하고, 어떻게 분할하고, 언제 병합할 것인가

```
힙 메모리 관리
├─ 1. 블록 표현
│  ├─ 블록 구조
│  └─ 메타데이터
├─ 2. 빈 공간 추적
│  ├─ implicit free list
│  └─ explicit free list
├─ 3. 할당/해제 정책
│  ├─ 블록 선택
│  ├─ 블록 분할
│  └─ 블록 병합
└─ 4. 단편화와 효율
   ├─ 내부 단편화
   └─ 외부 단편화
```

## 1. 블록 표현

- **블록 구조**: 힙을 block 단위로 나누는 방식
- **메타데이터**: block size, allocation bit 등 관리 정보
  - 그 외 header, footer, 정렬, prev/next 포인터 등

## 2. 빈 공간 추적

- **implicit free list**: 전체 블록을 순차적으로 보며 free block을 찾는 방식
- **explicit free list**: free block만 별도로 연결해 관리하는 방식

## 3. 할당/해제 정책

- **블록 선택**: 어떤 free block을 사용할 것인가
  - first fit: 처음 만나는 충분히 큰 free block 선택
    - 속도: 중간(초반 탐색 비효율) / 메모리: 중간 
  - next fit: 이전 탐색 위치부터 계속 탐색
    - 속도: 좋음 / 메모리: 나쁨 <small>(논문에서 그렇다네용)</small>
  - best fit: 가장 크기가 잘 맞는 block 선택
    - 속도: 나쁨 / 메모리: 좋음
- **블록 분할**: 할당 시 남는 공간을 나눌 것인가
- **블록 병합**: 해제 시 인접 free block을 합칠 것인가
  - immediate coalescing: free 될 때 바로 병합
  - deferred coalescing: 나중에 필요할 때 병합

## 4. 단편화와 효율

위의 구조와 정책에 따라 메모리 효율이 달라지며, 그 결과 단편화가 발생한다.

- **내부 단편화**: 할당된 블록 내부에 남는 공간
- **외부 단편화**: free 공간이 흩어져 큰 연속 공간을 만들기 어려운 상태

# 그 외

- 실제 메모리 할당은 이것보다 훨씬 복잡하다. 동시성이나 최적화 등 고려해야 할 여러 요소가 많기 때문.
- 리눅스의 경우 블록에 푸터가 없다. 
- 이 글에서는 가상 메모리 위의 추상화된 환경을 기준으로 설명한다. 그 아래 계층은 운영체제가 추상화해 제공하는 영역이므로 여기서는 다루지 않는다. 
- `유저 코드 - 표준 라이브러리(malloc) - 시스템 콜 인터페이스(mmap, brk) - 커널/운영체제`으로 레벨을 구분하면서 생각해보면 더 명확하다. (이것들은 가상메모리와는 별개의 개념임)
